# One Flow

Notre équipe avons choisi le workflow _One Flow_ car elle permet de diviser des tâches spécifiques dans plusieurs branches.
C'est un flow souple et permet une meilleure cohésion. On peut se passer de la branche `develop`. 

Une fois qu'on est sûr de notre modification, mise à jour ou de notre fonctionnalité alors on peut l'intégrer dans notre branche principale, `master`.
